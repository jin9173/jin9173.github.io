해당 텍스트에 링크 추가

  1. 클릭 시 해당하는 /post-detail/어딘가/로 이동시키기
  1. Post List! 메뉴 클릭시 post_list.html로 이동시키기

url 네임 만들어주기

해당 경로 이름도 커스터마이징 가능

해당 urlpatterns에 있는 경로 이름을 변경할 경우 html 소스에 있는 경로의 이름도 변경해주어야 하기 때문에 번거로운 작업이 요구된다. 그러므로 해당 url이름을 변경하더라도 html소스의 경로 이름을 바꿀 필요 없도록 네임을 지정해줄 수 있다.

하지만, post-detail에는 NoReverseMatch 에러가 발생한다.

해당 에러는 Reverse for 'post-detail' with no arguments not found라는 메시지를 보여준다. 해석하면 'post-detail'의 Reverse함수에는 인수가 없다는 뜻이다. Reverse함수는 path('posts/', post_list, name="post-list") 에서 name="post-list"을 posts/로 번역 시켜준다. 즉, 해당 에러메시지는 post-detail/<int:pk>에서 <int:pk>를 번역할 수 없다는 뜻이다.

이를 shell-plus에서 실행해보기.

Reverse함수가 있는지 확인해보기.

Reverse 함수를 사용해보면 브라우저에서 보여졌던 에러메시지가 shell에서도 똑같이 뜬다.

Reverse 함수는 기본적으로 urlconf, args, kwargs, current_app이라는 매개변수를 가지고 있다. 그리고 post-detail의 url을 번역시키기 위해서는 pk라는 키워드 인자가 필요하다.

잘 나오는 것을 테스트해서 확인했으니 해당 소스를 수정해보자.
