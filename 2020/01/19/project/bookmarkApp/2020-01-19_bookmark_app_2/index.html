1.2 프로젝트 설정 파일 변경

bookmark/jin91/mysite/settings.py

  1. ALLOWED_HOSTS

  2. TEMPLATES

    : 템플릿 관련 사항. DIRS 항목은 프로젝트 템플릿 파일이 위치할 디렉터리를 지정한다. 템플릿 파일을 찾을 때, 프로젝트 템플릿 디렉터리는 애플리케이션 템플릿 디렉터리보다 먼저 검색한다.

  3. DATABASES

    : 프로젝트에 사용할 데이터 베이스 엔진을 설정. 장고는 default로 SQLite3 데이터베이스 엔진을 사용하도록 설정되어 있다. (다른 데이터베이스 엔진으로 변경 가능)

  4. TIME_ZONE

    : 타임 존 지정. 처음에는 세계표준시(UTC)로 되어 있는데, 한국시간을 변경한다.

  5. STATIC_URL

    : 정적 파일에 관한 설정.

  6. MEDIA_URL, MEDIA_ROOT

    : 미디어 관련 사항을 지정. (파일 업로드 기능을 개발할 때 필요)

  7. LANGUAGE_CODE

    : Admin 사이트 화면의 메뉴 및 설명 등을 한글로 표기


1.3 기본 테이블 생성

bookmark/jin91

아직 데이터베이스 테이블을 만들지도 않았는데 해당 명령을 입력하여 기본 테이블을 생성하는 이유는 장고는 모든 웹 프로젝트 개발 시 반드시 사용자와 그룹 테이블 등이 필요하다는 가정 하에 설계되었기 때문에 명령을 실행한다.

명령을 실행하면, migrate 명령에 대한 로그가 보이고, 실행 결과로 SQLite3 데이터베이스 파일인 db.sqlite3 파일이 생성된 것을 확인할 수 있다.

2. 코딩하기

2.1 테이블 정의 (모델)

: 북마크 앱에는 테이블 하나만 필요

bookmarkApp/bookmark/models.py

(설명)

  1. title 컬럼은 공백(blank)값을 가질 수 있다.

  2. URLField() 필트 클래스의 첫 번째 파라미터인 'TITLE' 은 url 컬럼에 대한 별칭(verbose_name)

  3. __str()__ 함수는 객체를 문자열로 표현할 때 사용.

    (장고에서 모델 클래스의 객체는 테이블에 들어있는 레코드 하나를 의미한다. 해당 함수를 정의하지 않으면 후에 쓰게 될 Admin 사이트나 장고 셸 등에서 테이블의 레코드명이 제대로 표현되지 않는다. )

2.4 URLconf

: 해당 앱은 간단하므로 하나의 파일에 코딩할 것이다. Admin 사이트를 포함하여 3개의 url이 필요하다.

클래스형 뷰가 간단한 경우는 views.py 파일에 코딩할 필요없이, URLconf에서 뷰 및 뷰 처리에 필요한 파라미터를 모두 지정할 수 있다. 하지만 장고의 MTV 원칙을 따르기 위해서는 간단하더라도 views.py에 코딩하는 것이 향후 확장성과 모듈의 임포트 관계를 단순하게 유지할 수 있다.


2.5 뷰와 제네릭뷰

제네릭뷰 (generic view)

: 함수형 뷰를 shortcuts.render 를 사용하여 표현할 수 있지만 장고의 제네릭뷰를 사용하면 최소한 그 뷰가 어떤 모델을 이용할 것인지만 지정해주면 나머진 모두 상위 클래스(generic view)가 모든걸 알아서 해주기 때문에 코드의 흐름이 명료하고 코드가 읽기 쉬워질 수 있다.
