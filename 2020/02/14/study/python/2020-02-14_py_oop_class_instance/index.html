클래스와 인스턴스

인스타그램과 같은 SNS를 만드려고 할 때, User를 나타내는 객체가 필요하다.

User 객체
  • 속성: 이름, 이메일주소, 비밀번호, 팔로우 목록, 팔로워 목록
  • 행동
    • 자기소개하기
    • 다른 유저를 팔로우하기

모든 유저 객체들은 위와 같은 속성과 행동들을 가지고 있다. 이를 유저 객체의 틀을 정했다고 말한다. 객체의 틀이 있으면 이 틀을 이용하여 객체를 계속 만들 수 있다.

파이썬에서는 이런 객체의 틀을 클래스라고 하고 틀로 만든 결과물을 인스턴스라고 한다.

인스턴스와 객체의 차이
  • 객체(Object): 소프트웨어 세계에 구현할 대상
  • 클래스(Class): 이를 구현하기 위한 설계도
  • 인스턴스(Instance): 이 설계도에 따라 소프트웨어 세계에 구현된 실체 (인스턴스 < 객체)

객체는 현실의 대상과 비슷하여 상태나 행동 등을 가지지만, 소프트웨어 관점에서는 그저 콘셉에 불과하다. 소프트웨어 객체를 구현하기 위해서는 콘셉 이상으로 많은 것들을 사고하여 구현해야 하므로, 이를 위한 설계도로 클래스를 작성한다. 설계도를 바탕으로 객체를 소프트웨어에 실체화하면 그 것이 인스턴스가되고, 이 과정을 인스턴스화(Instantiation)라고 한다. 그리고 실체화된 인스턴스는 메모리에 할당된다.

User클래스에 속성 추가

인스턴스 변수 사용

여기서 주목해야 할 점은 각 User 인스턴스들은 같은 이름의 속성들을 가지고 있어도 서로 다른 속성값들을 가지고 있다. 즉, user1, 2, 3가 속성을 공유하는게 아니라 각자 개인적으로 가지고 있는 것이다. 이렇게 인스턴스가 개인적으로 가지고 있는 속성을 인스턴스 변수라고 한다. name, email, password는 모두 인스턴스 변수이다.


인스턴스 메소드

객체의 행동을 나타내는 함수

메소드의 종류

  1. 인스턴스 메소드

    : 인스턴스 변수를 사용하거나, 인스턴스 변수에 값을 설정하는 메소드

  2. 클래스 메소드

  3. 정적 메소드

인스턴스 메소드의 특별한 규칙

say_hello() 메소드를 보면 some_user라는 파라미터를 넘겨주어야 정상적으로 작동하는게 이론이지만 넘겨주지 않아도 오류없이 잘 실행된다. 그 이유는 인스턴스 메소드의 특별한 규칙때문이다.

User.say_hello(user1)은 클래스에서 메소드를 호출했고, user1.say_hello()는 인스턴스의 메소드를 호출했다.

인스턴스의 메소드를 호출하면, user1 인스턴스가 say_hello()의 첫번째 파라미터로 자동으로 전달된다. 즉, 파라미터를 따로 써줄 필요가 없다. 또한, User.say_hello(user1)과 user1.say_hello()는 다르게 생겼지만 완전히 똑같다고 볼 수 있다.

그렇다면 user1.say_hello()에 파라미터를 넣으면 어떻게 될까?

에러메시지를 살펴보면, say_hello()는 파라미터를 1개만 받는데 2개가 넘어왔다고 나온다. 즉, user1.say_hello()는 이미 user1이 파라미터로 넘어간 상태인데 user1.say_hello(user1)을 하게되면 추가로 user1이 하나 더 넘겨졌기 때문에 에러가 발생한 것이다. (User.say_hello(user1, user1) == user1.say_hello(user1))

로그인 메서드 추가

self

인스턴스 메소드를 호출하면 이 인스턴스가 메소드의 첫번째 파라미터로 자동전달된다. 즉, 인스턴스 메소드를 정의할 때는 항상 첫번째 파라미터로 호출하는 인스턴스의 자기 자신을 받는다. 위의 코딩은 그 파라미터를 some_user로 지정해주었다.

파이썬에서는 인스턴스 메소드의 첫번째 파라미터의 이름을 self로 쓰라고 권장한다. 즉, some_user는 self로 변경 가능하다. (다른 단어를 써도 실행에는 문제가 없지만 파이썬 세계의 약속이기 때문에 인스턴스 메서드의 첫번째 파라미터의 이름은 꼭 self로 쓰길 바란다.)

인스턴스 변수와 같은 이름을 갖는 파라미터

check_name()의 리턴값으로 self.name == name이 온다. 이는 인스턴스 변수와 파라미터의 이름이 똑같은데 문제가 없다. name은 check_name()함수 안에서 사용하는 파라미터의 이름일 뿐이고, self.name의 name은 self(인스턴스) 안에 있는 name이라는 변수(속성)이기 때문에 해당 코드는 문제가 없다. 즉, 육안상 같은 변수 같지만 사실은 전혀 다른 변수인 것이다.


__init__ 메소드

: 이름 앞뒤로 _가 두개씩 있는 메소드를 magic method 또는 special method라고 한다. 우리말로는 특수 메소드라고 하는데 이는 특정 상황에서 자동으로 호출되는 메소드이다.

특수 메서드는 양 옆으로 언더스코어가 두개씩 있기 때문에 double underscore를 줄인 dunder 메서드라고 부르기도 한다. 그러므로 init은 던더 init, str은 던더 str이라고 불린다.

결국 init메서드를 사용하면 인스턴스 생성과 인스턴스 변수 초기값 설정을 한 줄에 코딩할 수 있다. 그렇기 때문에 클래스에는 보통 init메서드를 꼭 작성한다.


__str__ 메소드

: print함수를 호출할 때 자동으로 호출되는 메서드이다. 즉, 어떤 인스턴스를 출력하면 던더 에스티알 메서드의 return값이 출력된다는 것이다.

인스턴스를 출력할 때마다 정보가 이런식으로 나오면 큰 의미가 없을 것이다. 원하는 내용이 출력되도록 함수를 만들어보자.


클래스 변수

인스턴스 변수란 인스턴스 자신만의 속성을 나타내는 것을 말한다. 그렇다면 여러 인스턴스들이 공유하는 속성은 클래스 변수라고 한다.

예를 들어, SNS의 User클래스에 User 인스턴스의 총 개수(count)를 나타내는 속성을 만들고 싶다. 이 속성은 특정 인스턴스가 갖고있는 값이 아니라 서로 공유하는 값이다. 어떤 User 인스턴스라도 똑같은 값을 가지고 있어야 한다. 파이썬에서 이러한 속성을 클래스 변수라고 한다. 즉, 클래스 변수는 같은 클래스의 인스턴스들이 서로 공유하는 값이다.

그렇다면 클래스 변수 count가 User 인스턴스 개수를 정확히 나타내도록 하려면 User 인스턴스가 생성될 때마다 count의 값을 1씩 늘리면 된다. User 인스턴스가 생성될 때마다 __init__ 메서드가 실행되기 때문에 __init__ 메서드 안에 count의 값을 늘려주면 된다.

클래스 변수 활용

같은 이름의 클래스 변수와 같은 이름의 인스턴스 변수가 있으면 인스턴스 변수가 읽어진다.

클래스 변수 인스턴스 변수
User.count user1.count

이런식으로 헷갈릴 수 있기 때문에 클래스 변수의 값을 설정할 때는 “클래스 이름”으로만 코딩해야 한다.

한 클래스의 모든 인스턴스가 공유하는 속성이라면 클래스 변수로 코딩한다. 그리고 클래스 변수는 클래스이름.클래스변수이름이나 인스턴스이름.클래스변수이름으로 값을 읽을 수 있지만 클래스 변수 값을 설정할 때에는 꼭 클래스이름.클래스변수이름으로만 코딩할 것을 권장한다.
