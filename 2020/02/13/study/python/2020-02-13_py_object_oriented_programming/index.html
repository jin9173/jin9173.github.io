객체(Object)

: 속성과 행동으로 이루어진 존재

세상의 모든 것들은 객체이다. 현실에 존재하든, 가상으로 존재하든 속성과 행동을 떠올릴 수 있다면 모두 객체이다.

예를 들어, 자동차 객체의 속성에는 높이, 의자 개수, 색깔 등이 있다. 또한 자동차는 시동이 걸리는 행동과 엑셀을 밟으면 앞으로 나아가는 행동을 수행한다. 이런 식으로 자동차에는 속성과 행동이 있기 때문에 객체로 볼 수 있다.

또 다른 예시로 인스타그램의 유저들을 객체로 바라봐보자. SNS 유저의 속성으로는 이메일주소, 비밀번호, 친구 목록과 같은 것들이 있을 것이다. 그리고 게시물에 좋아요를 누르는 행동과 친구 추가하기와 같은 행동이 있다. SNS 유저들은 실제 물리적으로는 존재하지 않지만 속성과 행동이 있기 때문에 객체라고 볼 수 있다.


객체 지향 프로그래밍(Object Oriented Programming)

: 프로그램을 여러 개의 독립된 객체들과 그 객체들 간의 상호작용으로 파악하는 프로그래밍 접근법

즉, 객체지향 프로그래밍이란 객체들과 객체들 간의 소통으로 바라보는 것이다.

예를 들어, 객체지향 프로그래밍을 이용하여 총게임를 만든다고 가정해보자.

총게임에 필요한 객체들
  • 게임 캐릭터 객체
  • 총 객체
  • 총알 객체
해당 객체에 필요한 속성들과 행동들

  • 게임 캐릭터 객체

    • 속성: ID, 현재 사용중인 총, 체력, 목숨
    • 행동
      • 현재 사용중인 총을 발사한다
      • 달린다
      • 체력이 0이 되면 죽는다

  • 총 객체

    • 속성: 모델명, 무게, 장전된 총알 개수
    • 행동: 총알을 발사한다

  • 총알 객체

    • 속성: 공격력
    • 행동: 총알을 맞은 (상대) 캐릭터의 체력을 공격력만큼 깎는다
해당 객체들은 어떻게 소통(상호작용)해야 할까?

  1. 게임 캐릭터 객체가 총 객체에게 발사하라고 신호를 보낸다.

  2. 총 객체는 장전된 총알 중 하나를 발사한다.

    (이 때, 자신의 속성 중 장전된 총알 개수에서 1을 뺀다.)

  3. 발사된 총알이 다른 게임 캐릭터의 객체에 닿으면 이 총알은 그 객체에게 해당 공격력만큼 그 객체의 체력을 깎으라는 신호를 보낸다.

  4. 총알의 신호를 받은 상대 게임 캐릭터 객체는 그 총알의 공격력만큼 자신의 체력을 깎는다.

    (이 때, 깎고난 후의 체력이 0이하라면 죽어야 한다. 만약 죽으면 목숨 수에서 1을 빼야한다.)

이와 같은 과정들은 ‘객체지향적으로 설계를 했다’라고 표현할 수 있다. (모델링 modeling)


객체 지향 프로그래밍으로 프로그램을 만들려면,

  1. 프로그램에 어떤 객체들이 필요할지 정한다.
  2. 객체들의 속성과 행동을 정한다.
  3. 객체들이 서로 어떻게 소통할지 정한다.
