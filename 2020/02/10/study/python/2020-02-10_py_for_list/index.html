while문을 for문으로 바꾸기

for문을 사용하여 while문의 불필요한 부분들을 지우고 직관적이게 코드를 짜보자.


range()

  • 간편하고 깔끔 (굳이 리스트를 만들지 않아도 동일한 효과 구현 가능)
  • 효율적인 메모리 공간 사용 (1부터 100까지의 리스트를 쓰면 파이썬에서 그 만큼의 공간을 마련해야 하지만 range함수를 쓰면 1의 값을 쓰고 버리고 2의 값을 쓰고 버리고 하는 식으로 메모리 공간을 아낄 수 있다)

파라미터가 2개 있는 range()

: range(n, m)은 n부터 m-1까지의 수들을 의미

파라미터가 1개 있는 range()

: range(m)은 0부터 m-1까지의 수들을 의미

파라미터가 3개 있는 range()

: range(n, m, s)은 n부터 m-1까지의 수 중 간격이 s인 수들을 의미


엘리어싱 (Aliasing)

x는 [2, 3, 5, 7, 11]이 나오고 y는 [2, 3, 4, 7, 11]이 나올 줄 알았는데 결과과 예상과 다르다. y의 원소만 바꾸었을 뿐인데 x와 y가 똑같이 [2, 3, 4, 7. 11]로 변한 이유는 엘리어싱이라는 개념 때문이다.

코드의 첫번째 줄에서는 [2, 3, 5, 7, 11] 리스트에 x라는 이름표를 달아준다.

두번째 줄에서 y = x라는 명령에 의해, 그 같은 리스트에 y라는 이름표를 달아준다.

세번째 줄에서 리스트 y의 인덱스 2의 값을 4로 바꿔주면, 리스트는 [2, 3, 4, 7, 11]이 된다.

그런데 바뀐 [1, 2, 4, 7, 11]이라는 리스트가 y라는 이름표 뿐만 아니라, x라는 이름표도 갖고 있다. 그래서 x와 y를 출력하라는 명령이 있을 때, 모두 같은 [2, 3, 4, 7, 11]이 출력되는 것이다. 두 변수가 서로 이름은 다르지만, 사실 같은 메모리 주소에 있는 값을 가리키고 있기 때문에 동일한 리스트가 출력된 것이다. 즉, y는 x의 Alias(가명)이라고 할 수 있다.


리스트 복사 list()

리스트의 원본을 그대로 둔 채, 리스트의 복사본만 바꾸고 싶은 경우


리스트와 문자열

공통점

인덱싱 (Indexing)

두 자료형은 공통적으로 인덱싱 가능

for 반복문

두 자료형은 인덱싱이 가능하기 때문에 for 반복문에서도 활용 가능 (이터러블 객체)

슬라이싱 (Slicing)

두 자료형은 공통적으로 슬라이싱 가능

덧셈 연산 +

두 자료형에게 모두 덧셈은 연결하는 연산이다

len()

두 자료형은 모두 길이를 재는 len함수를 쓸 수 있다

차이점

Mutable(수정 가능 - 리스트) VS Immutable(수정 불가능 - 문자열, 숫자, 불린)

리스트는 데이터를 바꿀 수 있지만 문자열은 바꿀 수 없다.
